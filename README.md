## 核心价值

SmartFile 解决的是一个几乎所有人都面临的问题：**文件命名混乱**。无论是 `IMG_20241215_143022.jpg`、`截屏2024-12-20 18.36.08.png`、`新建文档 (2).docx`，还是 `录音 (3).m4a`，这些文件名完全无法帮助你回忆文件内容。SmartFile 通过本地运行的 AI 模型，自动分析文件内容并生成语义化的文件名和归档结构。

| 痛点 | SmartFile 方案 |
|------|---------------|
| 截图命名毫无意义（截屏2024-12-20...） | AI 识别截图内容，重命名为 `2024-12-20_产品原型评审截图.png` |
| 照片只有相机编号（IMG_20241215...） | 视觉模型理解场景，重命名为 `2024-12-15_办公室团建合影.jpg` |
| 文档名称混乱（新建文档(2).docx） | 解析文档内容，重命名为 `2024-12-16_Q4季度总结报告.docx` |
| 录音文件无法区分（录音(3).m4a） | 基于元数据推断，重命名为 `2024-12-18_客户需求沟通录音.m4a` |
| 文件散落各处无归档 | 自动按语义分类到 `照片/活动`、`文档/合同` 等目录 |

## 功能特性

### 多模态内容识别

SmartFile 不仅能"读懂"文本文档，还能"看懂"图片内容和"听懂"录音文件，通过多模态 AI 模型实现全面的文件理解。

- **图片理解**：使用通义千问 Qwen2.5-VL 视觉模型，识别照片场景、截图内容、设计稿主题
- **文档解析**：读取 TXT、Markdown、CSV 等文本文件的实际内容，提取主题和关键信息
- **音频推断**：基于文件名和元数据智能推断录音内容类型（会议、电话、备忘等）

### 自定义语义规则链

用自然语言下达归档指令，AI 自动理解并执行：

```
"把所有合同按年份和金额归类"
→ 文档/合同/2024/大额 | 文档/合同/2024/小额

"照片按拍摄地点分类，工作和生活分开"
→ 照片/工作/办公室 | 照片/生活/公园
```

### 本地化隐私部署

所有 AI 分析完全在本地完成，文件数据不会上传到任何服务器：

- 使用 Ollama 运行端侧模型，无需云端 API
- 模型下载后可完全离线使用
- 开源模型（通义千问 Qwen），代码完全可审计

### 智能模型推荐

根据你的 Mac 硬件配置，自动推荐最适合的模型组合：

| 设备配置 | 文本模型 | 视觉模型 | 适用场景 |
|---------|---------|---------|---------|
| 8GB RAM | qwen2.5:1.5b (1.0GB) | qwen2.5vl:3b (2.5GB) | 日常文件整理 |
| 16GB RAM | qwen2.5:7b (4.5GB) | qwen2.5vl:3b (2.5GB) | 高质量分析 |
| 32GB+ RAM | qwen2.5:14b (9.0GB) | qwen2.5vl:7b (5.0GB) | 专业级分析 |

## 快速开始

### 前置要求

- **macOS** 12.0 或更高版本
- **Node.js** 18+ 和 **pnpm**
- **Ollama** (本地 AI 模型运行引擎)

### 一键安装

```bash
# 克隆项目
git clone <your-repo-url> smartfile
cd smartfile

# 运行安装脚本（自动检查环境、推荐模型、安装依赖）
chmod +x scripts/setup.sh
./scripts/setup.sh
```

### 手动安装

```bash
# 1. 安装 Ollama
# 访问 https://ollama.com/download 下载安装

# 2. 下载 AI 模型（根据你的内存选择）
ollama pull qwen2.5:7b        # 文本分析模型 (16GB+ 推荐)
ollama pull qwen2.5vl:3b      # 视觉理解模型

# 3. 安装项目依赖
pnpm install

# 4. 安装 Electron
cd electron
npm install electron electron-builder --save-dev
cd ..
```

### 启动应用

```bash
# 开发模式
pnpm run dev                   # 启动 Web 开发服务器 (localhost:3000)
cd electron && npm run dev     # 启动 Electron 桌面窗口

# 构建 macOS 应用
pnpm run build                 # 构建前端
cd electron && npx electron-builder --mac  # 打包 .dmg
```

## 工作原理

SmartFile 的文件分析流程分为四个步骤：

1. **扫描**：读取指定文件夹中的所有文件，提取文件名、类型、大小、日期等元数据
2. **分析**：根据文件类型选择合适的 AI 模型进行内容理解
   - 图片文件 → 视觉模型（qwen2.5vl）识别场景、文字、主题
   - 文档文件 → 文本模型（qwen2.5）解析内容、提取关键信息
   - 音频文件 → 基于元数据和文件名推断内容类型
3. **命名**：AI 生成语义化的文件名（格式：`日期_主题描述.扩展名`）和归档目录建议
4. **确认**：用户逐一或批量确认重命名方案，SmartFile 执行实际的文件重命名和移动操作

## 常见问题

**Q: Ollama 安装后无法启动？**

确保 Ollama 服务在后台运行：`ollama serve`。如果端口被占用，检查是否有其他 Ollama 实例在运行。

**Q: 模型下载速度很慢？**

模型文件较大（1-20GB），下载速度取决于网络环境。建议使用稳定的网络连接，或在网络条件好的时候提前下载。

**Q: 8GB 内存的 Mac 能用吗？**

可以。推荐使用 qwen2.5:1.5b 文本模型（仅需 1GB），基本的文件重命名功能可以正常使用。视觉模型（图片理解）建议 16GB 以上内存。

**Q: 文件数据会上传到云端吗？**

不会。SmartFile 的所有 AI 分析完全在本地完成，通过 Ollama 运行端侧模型。文件数据不会离开你的电脑。

**Q: 支持哪些文件类型？**

图片（jpg, png, gif, webp, heic 等）、文档（pdf, docx, xlsx, txt, md, csv 等）、音频（mp3, m4a, wav 等）、视频（mp4, mov 等）。
